PDF Generator API · Sales

Design your own quotation & sale-order PDFs

Generate branded PDFs from quotations and sale orders with pdfgeneratorapi.com — designed visually in Odoo, no QWeb required.

What this bridge adds

This module wires the PDF Generator API into sale.order — quotations and confirmed sale orders.

Generate button, where you work

A Generate custom PDF button on the sale-order form (with a Generate and Download option), plus a toolbar action on the Quotations and Sales Orders lists to render a whole selection at once.

Ready-to-edit field dataset

A starter Quotation / Sale Order dataset maps order fields — number, dates, validity, customer reference, partner and company details, order lines, taxes and totals — to your template placeholders. Editable from the UI, no code.

Part of the PDF Generator API suite

This is a bridge module. Installing it automatically installs the PDF Generator API framework module, which provides the API client, the visual template editor embedded in Odoo, per-model field datasets, coverage diagnostics, and the async job machinery.

Connect your workspace once under Settings → PDF Generator API (API key, secret and workspace identifier), then click Generate custom PDF on any quotation or sale order. See the PDF Generator API app for the full feature list, hosting compatibility and privacy details. A pdfgeneratorapi.com account is required — PDF rendering happens on their service.

Key features

Drag-and-drop template editor

Design templates visually in the pdfgeneratorapi.com editor embedded in Odoo — no QWeb, no XML overrides. Preview against a real quotation while you design.

Smart field mapping

Datasets map Odoo fields to placeholders, including repeated sections like order lines and taxes. Editable from the UI with a drag-and-drop field palette — no code.

Async batch dispatch

Select many orders in a list and generate PDFs for all of them in one click. Per-record async jobs with a signed webhook callback keep Odoo responsive.

Multi-company & secure

Per-company API credentials with a global fallback, JWT-signed requests (HS256), webhook signature verification, and secret redaction in logs. Every generation is logged to the record's chatter.
